## What Changed

### 1) Better image import quality and filtering

In `app/api/management/mini-gt-import/route.ts`:

- The import now accepts more image path types (not just one old path).
- The system now **prioritizes better image sources first**.
- It removes duplicate image links and keeps a clean order.
- It now preserves source order inside each priority bucket (not alphabetical), so image order is more stable.
- It only scans the **main product section** of the page, so unrelated "related products" images are avoided.
- Result: image import is more accurate and gives better quality images.

### 2) Mini GT collection page got major usability upgrades

In `app/collection/mini-gt/page.tsx`:

- Added better data typing/structure to make data handling safer.
- Added in-page import flow:
  - import modal
  - draggable modal window
  - series + product URL input
  - actions: "Import + Done" and "Import + Add New"
- Added per-card helper for missing images:
  - small "Import Image" button on cards with no image
  - opens import modal with series prefilled
- Added infinite scroll style loading ("load more while scrolling") for smoother browsing.
- Added floating list status indicator:
  - "Showing X / Y" always visible during scroll
- Added touch swipe support in the image carousel (mobile-friendly).
- Added refresh image button inside preview modal (near item_no + scale).
- Refetch in modal is now scoped to **current item/card only**.
- Added image cache-busting support for latest preview refresh.
- Added touch swipe support in the image carousel (mobile-friendly).
- Improved visual loading behavior:
  - skeleton while image is unresolved
  - clearer fallback "No image yet" state
- Cleaned up old center overlay text that caused duplicate numbering appearance.

### 3) Carousel and dot behavior updates

In `app/collection/mini-gt/page.tsx`:

- This keeps dot count behavior closer to expected baseline.
- Dot count still adjusts automatically when actual available images are more/less.

## User Impact
