## What Changed

### 1) Fixed catalog link not landing on the correct brand tab

In `components/purchase/purchase-grid.tsx` and `components/collection/collection-grid.tsx`:

- Previously the catalog link only sent `?search=<term>`, so the catalog always opened on the Mini GT tab regardless of which brand the car belonged to.
- Now the car's brand name is also sent as `?brand=<name>` in the URL.
- The catalog reads this and opens directly on the correct brand tab.
- Example: clicking the link for a Tarmac Works car now lands on the Tarmac Works tab, not Mini GT.

In `app/catalog/page.tsx` and `app/catalog/components/CatalogClient.tsx`:

- Catalog page now reads the new `brand` query param.
- `CatalogClient` accepts `initialBrand` and uses it to set the starting brand tab instead of always defaulting to Mini GT.

### 2) Expanded catalog link to non-Mini GT brands

In `components/purchase/purchase-grid.tsx` and `components/collection/collection-grid.tsx`:

- Previously the catalog link icon only appeared for Mini GT cars with a valid item number.
- Now it also appears for any car that has images stored in the remark field (e.g. Tarmac Works, Inno64).
- Uses the shared `hasCatalogImages()` helper to decide whether to show the icon.

### 3) Image cache busting after reload

In `app/catalog/components/CatalogClient.tsx`:

- When the user clicks the reload button, a timestamp (`imageReloadBust`) is now recorded.
- This timestamp is passed to the detail sheet so images are forced to re-fetch with a new cache-bust version instead of serving a stale cached copy.

In `app/catalog/components/CardDetailSheet.tsx`:

- Accepts `imageReloadBust` and applies it to all image URLs in the detail view.
- Failed-URL tracking now strips the cache version before storing, so reloading clears the failed state correctly.

In `app/catalog/components/DiecastCard.tsx`:

- Same fix as above â€” failed-URL tracking now strips cache version before storing so cards recover properly after a reload.

### 4) `updated_at` bumped on image upload and import

In `app/api/management/collection-image-upload/route.ts` and `app/api/management/mini-gt-import/route.ts`:

- When images are saved to a collection, the `updated_at` timestamp on the collection record is now also updated.
- This means the catalog automatically picks up the correct cache version for newly uploaded images without needing a manual workaround.

### 5) Improved image cache headers

In `app/api/catalog/image/[...path]/route.ts`:

- Added `stale-while-revalidate=604800` to the `Cache-Control` header.
- Browsers can now serve a cached image immediately while checking for a newer version in the background, making catalog browsing feel faster.

### 6) Consolidated image utility helpers

In `lib/collection-images.ts`:

- Moved several duplicated helper functions from `app/catalog/page.tsx` into the shared library:
  - `extractRemarkImageUrls` â€” parses image URLs from the remark field
  - `getMiniGtSeriesImageUrls` â€” generates slot-based image URLs for Mini GT series
  - `hasCatalogImages` â€” checks whether a collection item has any catalog images
  - `buildCatalogSearchTerm` â€” picks item number or model name as the catalog search term
  - `buildCatalogItemImageUrls` â€” builds the full merged image URL list for a collection item
  - `stripImageCacheVersion` â€” removes the `?v=` cache-bust param from a URL
  - `appendImageCacheVersion` â€” adds a `?v=` cache-bust param to a URL
- These are now reusable across the catalog, grids, and API routes.

### 7) Fixed "Import From URL" button style

In `app/management/image-import/page.tsx`:

- The "Import From URL" button was using `variant="secondary"` which made it look different from the "Upload Images" button.
- Changed to use the default (primary) button style to match.
- Added a `Globe` icon next to the label, consistent with how the Upload button shows an icon.
