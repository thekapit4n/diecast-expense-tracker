## What Changed

### 1) Purchase price now accepts RM 0.00

In `app/purchase/add/page.tsx` and `components/purchase/edit-purchase-modal.tsx`:

- The price per unit field previously rejected any value of `0.00`.
- This was a problem because some items are obtained for free — mystery box pulls, gifts from friends, event freebies, etc.
- The Zod validation rule was changed from `num > 0` to `num >= 0` in both the Add Purchase page schema and the Edit Purchase modal schema.
- The validation error message was also updated to match the new rule:
  - Before: `"Price must be a positive number"`
  - After: `"Price must be 0 or a positive number"`
- The HTML input already had `min="0"` set, so no further UI changes were needed.

## User Impact

- Users can now record a purchase with price RM 0.00.
- Mystery box items, gifts, and event freebies can be properly logged without workarounds.
- Validation still rejects empty or non-numeric values as before.
