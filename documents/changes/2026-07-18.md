## What Changed

### 1) Chase photos now stored in their own subfolder

In `lib/collection-images.ts`, `app/api/catalog/image/[...path]/route.ts`, `app/api/management/collection-image-upload/route.ts`:

- Chase images are now saved to a dedicated `chase/` subfolder under the item's normal folder (e.g. `mini-gt/MGT00009/chase/1.jpg`) instead of mixing with the normal-release photos (`mini-gt/MGT00009/1.jpg`).
- The public image-serving route now understands this extra `chase` path segment for every brand, not just Mini GT.
- Photo numbering (1.jpg, 2.jpg, ...) is tracked separately per folder so chase and normal uploads don't clash or overwrite each other.

### 2) Image Import page — new "Is Chase?" option

In `app/management/image-import/page.tsx`:

- Added an "Is Chase?" dropdown next to Item Number (defaults to No).
- Choosing "Yes" routes the uploaded photos into the chase subfolder; Storage Preview and the Upload Result banner now show the `/chase` path so it's clear where the files landed.

### 3) Catalog — chase and normal now shown as separate cards (when both owned)

In `app/catalog/page.tsx`, `lib/collection-images.ts`:

- If you've bought both a normal unit and a chase unit of the same model, the catalog grid now shows **two tiles** — one tagged "Chase", one plain — instead of merging them into one card.
- If you only own one version (chase-only, normal-only, or haven't bought it at all yet), it still shows as a single card — no empty "ghost" placeholder appears for a version you don't have.
- Each tile only pulls photos from its own folder (chase tile → chase photos only, normal tile → normal photos only), so the pictures never mix between the two.
- Both tiles keep the same name / item number / brand / scale — only the badge, price history, and photos differ, so it's obvious they're the same real-world item in two versions.

### 4) Catalog detail sheet — clearer chase price indicator

In `app/catalog/components/CardDetailSheet.tsx`:

- Each purchase row in Purchase History is now tinted red when that specific unit was a chase purchase (same visual treatment the amber "Pre-order" tint already had).
- A small "Chase" label now sits right next to the shop name, and the price itself turns red for that row — so it's immediately obvious which price belongs to the chase unit when a model has several purchase entries.
- Removed the old bottom "Chase Unit" strip, which could get visually cut off or hidden behind other on-screen elements.

### 5) Add Purchase — removed duplicate Mini GT image import

In `app/purchase/add/page.tsx`:

- Removed the old "Mini GT Image Import" box (product URL paste + import button) and all its related state/logic — it duplicated what the Image Import page already does, and had no support for chase photos.
- Added a short note under "Collection Information" pointing to the Image Import page for uploading photos, including chase variants.

## User Impact

- Chase and normal versions of the same diecast no longer share confusing mixed photos — each version has its own set now.
- Collectors who own both versions of a model can browse them as two clearly separate catalog entries, each with the correct price history and photos.
- Uploading chase photos is a deliberate choice on the Image Import page instead of guessing which folder they land in.
- Purchase history makes it obvious at a glance which price was paid for the chase unit vs the normal one.
- Add Purchase page is simpler — one clear place (Image Import) for all photo uploads, no duplicate flow to maintain.

## Follow-up fixes (same day, after first pass)

Found while using the Add Purchase form live after the commit above — file: `app/purchase/add/page.tsx`.

### 6) Fixed collapsible sections needing two clicks to open the first time

- "Pre-order Tracking" and "Additional Information" on the purchase form are collapsed by default. `isSectionCollapsed` treats *anything that isn't explicitly `false`* as collapsed (including the unset default), but `toggleSection` was flipping the raw stored value — `undefined` negates to `true`, which still reads as collapsed. So the very first click on either section silently did nothing; only the second click actually opened it. Every click after that worked normally, since by then the state held an explicit `true`/`false`.
- Fixed by having `toggleSection` flip the *computed* collapsed state (via `isSectionCollapsed`) instead of the raw stored value, so the first click opens the section immediately like every click after it.
