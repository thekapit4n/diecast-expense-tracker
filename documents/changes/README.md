# Daily Changes Log

Use this folder to record project updates day by day in simple English.

## File Naming Rule

Always create one file per day using this format:

- `YYYY-MM-DD.md`

Example:

- `2026-04-26.md`

## What to Write Each Day

Keep it simple so non-technical readers can understand.

Suggested sections:

1. Quick overview
2. What changed
3. User impact
4. Notes for next time

## Daily Template

Copy this into each new daily file:

```md
# Changes Log - YYYY-MM-DD

This note records all work done today in simple language.

## Quick Overview

- List the main updates in 2-5 bullets.

## What Changed

### 1) [Area or feature]

- Explain in plain language what was changed.
- Explain why it was changed (if relevant).

### 2) [Area or feature]

- Explain in plain language what was changed.

## User Impact

- Explain what users/team gain from these updates.

## Notes for Next Time

- Add anything important for follow-up.
```
