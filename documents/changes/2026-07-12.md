## What Changed

### 1) New Pre-order Tracker page (`/purchase/preorders`)

In `app/purchase/preorders/page.tsx` and `components/purchase/preorder-tracker.tsx`:

- New page that lists every purchase linked to a PO order.
- Groups items under their PO deal header (reference, seller, channel, ETA, close date).
- Shows lifecycle stages: awaiting production → ready for pickup → collected.
- Summary cards count items in each stage and show total outstanding balance.
- Search box filters by model name, item number, PO reference, or shop name.
- Channel icons for Facebook, WhatsApp, and Instagram.
- External link to the original PO announcement post when `source_link` is saved.
- Checkbox selection lets you bulk-mark ready items as collected in one action.
- Refresh button reloads tracker data without leaving the page.

### 2) PO order header table and purchase tracking columns

In `supabase/migrations/20260712000000_add_po_order_tracking.sql`:

- New `tbl_po_order` table groups items bought under the same pre-order deal.
- Stores shared deal info once: seller, reference code, channel, ETA, close date, full-payment flag, order date, remark.
- `tbl_purchase` extended with:
  - `po_order_id` — link to the PO header
  - `variant_status` — regular / chase / combo / unknown (for random-chase reveals)
  - `ready_date` — when seller said item is ready
  - `pickup_deadline` — optional seller deadline (informational only)
  - `collected_date` — when you physically picked it up
  - `amount_paid` — running total actually paid so far (supports deposits/partials)

In `supabase/migrations/20260712010000_add_po_order_source_link.sql`:

- Added `source_link` on `tbl_po_order` for the original Facebook/WhatsApp PO post URL.

In `supabase/migrations/20260712020000_backfill_amount_paid.sql`:

- Existing paid purchases backfilled so `amount_paid = total_price` where `payment_status = 'paid'`.
- Makes dashboard totals accurate for old records, not just new ones.

## Follow-up fixes (same day, after first pass)

Found while testing the tracker live against real data — main files: `app/purchase/add/page.tsx`, `components/purchase/preorder-tracker.tsx`, `components/purchase/bulk-collect-dialog.tsx`, `components/purchase/po-status-modal.tsx`, `app/catalog/components/CardDetailSheet.tsx`.

### 8) Fixed the real cause of the MGT01177 duplicate collection

- In Add Purchase, picking an existing collection from **Collection Name** didn't fill in the **Item Number** box — two comboboxes were reading from separate state, so Item Number looked blank even though the right collection was selected underneath.
- That's what led to an accidental duplicate collection entry earlier — now Item Number, Brand, and Scale all fill in together the moment an existing collection is picked.
- Also checked the live database directly: confirmed no duplicate collections exist, and Dashboard "Total Spent" matches a manual recomputation from raw purchase data exactly (RM 32,887.08).

### 9) Pre-order Tracker — search, tabs, sort, packaging

- "All orders" list now has a search box (name or item number), **Pending / Collected** tabs (so the list doesn't grow forever with old finished orders), and a sort dropdown (Name or Pre-order date).
- Packaging type now shown next to variant on each tracked item row.

### 10) Payment method tracking completed

- Added a **Payment Method** field to the per-item Update modal and the bulk collect dialog (asks once per shop visit, applies to all selected items) — covers the "pay on pickup" flow that had no way to record how you paid.
- Added an explicit **"Not specified"** option to Payment Method everywhere (Add Purchase, Edit Purchase, Update modal) so it can be intentionally left blank instead of getting stuck on the first value you pick.
- Marking an item "Collected" now consistently flips Payment Status to Paid regardless of whether you use the "Today" shortcut or pick a date manually.

### 11) Mobile layout bug in tracker dialogs

- The bulk-collect dialog and per-item Update modal were overflowing off the right edge of the screen on mobile (a CSS Grid default `min-width: auto` issue letting long item names push the dialog wider than the viewport).
- Fixed with constrained grid columns (`minmax(0,1fr)` + `auto`) so item names truncate with an ellipsis and price/total stay fully visible — verified on a 375px mobile viewport.

### 12) Catalog — copy title and item number

- In the catalog detail sheet, the title and item number badge are now click-to-copy to clipboard, with a toast confirmation. Useful for pasting into marketplace searches.
