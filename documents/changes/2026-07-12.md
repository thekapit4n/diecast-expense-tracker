## What Changed

### 1) New Pre-order Tracker page (`/purchase/preorders`)

In `app/purchase/preorders/page.tsx` and `components/purchase/preorder-tracker.tsx`:

- New page that lists every purchase linked to a PO order.
- Groups items under their PO deal header (reference, seller, channel, ETA, close date).
- Shows lifecycle stages: awaiting production → ready for pickup → collected.
- Summary cards count items in each stage and show total outstanding balance.
- Search box filters by model name, item number, PO reference, or shop name.
- Channel icons for Facebook, WhatsApp, and Instagram.
- External link to the original PO announcement post when `source_link` is saved.
- Checkbox selection lets you bulk-mark ready items as collected in one action.
- Refresh button reloads tracker data without leaving the page.

### 2) PO order header table and purchase tracking columns

In `supabase/migrations/20260712000000_add_po_order_tracking.sql`:

- New `tbl_po_order` table groups items bought under the same pre-order deal.
- Stores shared deal info once: seller, reference code, channel, ETA, close date, full-payment flag, order date, remark.
- `tbl_purchase` extended with:
  - `po_order_id` — link to the PO header
  - `variant_status` — regular / chase / combo / unknown (for random-chase reveals)
  - `ready_date` — when seller said item is ready
  - `pickup_deadline` — optional seller deadline (informational only)
  - `collected_date` — when you physically picked it up
  - `amount_paid` — running total actually paid so far (supports deposits/partials)

In `supabase/migrations/20260712010000_add_po_order_source_link.sql`:

- Added `source_link` on `tbl_po_order` for the original Facebook/WhatsApp PO post URL.

In `supabase/migrations/20260712020000_backfill_amount_paid.sql`:

- Existing paid purchases backfilled so `amount_paid = total_price` where `payment_status = 'paid'`.
- Makes dashboard totals accurate for old records, not just new ones.

### 3) Add Purchase — pre-order tracking section

In `app/purchase/add/page.tsx` and `components/ui/po-order-combobox.tsx`:

- Each purchase line now has an optional **Pre-order Tracking** section.
- Toggle "Is this a pre-order?" to link the item to a PO order.
- `PoOrderCombobox` lets you search existing PO orders or type a new reference.
- When creating a new PO order inline, you can set channel, ETA, close date, full-payment flag, source link, and order date.
- Variant status field for chase/combo/unknown outcomes.
- `amount_paid` is saved based on payment status (paid = full, partial = entered amount, unpaid = 0).
- PO reference is auto-suggested from item number + collection name when you turn tracking on.

### 4) PO status modal and bulk collect dialog

In `components/purchase/po-status-modal.tsx`:

- Modal to update a single tracked item: payment status, amount paid, payment date/method, ready date, pickup deadline, collected date.
- If payment is settled but no payment date exists, defaults to collected date (or today) so dashboard spending picks it up.

In `components/purchase/bulk-collect-dialog.tsx`:

- Confirms bulk collection for multiple items at the same shop.
- Shows item list and total RM.
- Asks how you paid (cash, QR, card, FPX) and marks all selected items paid in full + collected today.

### 5) Dashboard now uses `amount_paid` for spending totals

In `app/page.tsx`:

- Total spent, this-month spending, and top-brand breakdown now sum `amount_paid` instead of `total_price`.
- Reflects real money out — deposits count only what you've paid, not the full PO price until settled.

### 6) Delete collection and delete purchase (with confirmation)

In `components/ui/confirm-delete-dialog.tsx`:

- Reusable dialog that requires typing `DELETE` before a permanent action runs.

In `components/collection/collection-grid.tsx`:

- Right-click **Delete Collection** on the Name column.
- Shows how many purchase records will also be removed.
- Deletes all linked purchases, then the collection row.

In `components/purchase/purchase-grid.tsx`:

- Right-click **Delete Purchase** on the Collection Name column.
- Removes purchase + owned-unit detail record.
- If it was the last purchase for that collection item, removes the collection entry too.

### 7) Navigation and breadcrumbs

In `lib/theme/nav-config.ts`:

In `components/layout/page-breadcrumb.tsx`:

- Added breadcrumb label for the preorders route.

## User Impact

- Pre-orders from Facebook/WhatsApp sellers can now be tracked end-to-end in one place — no more spreadsheets for ETA, close dates, and pickup status.
- Multiple cars under the same PO deal share one header; you don't re-enter seller/ETA for every line item.
- Partial payments and deposits are reflected correctly in dashboard spending.
- Bulk "mark as collected" saves time when picking up several items at one shop.
- Deleting wrong entries is possible again, but guarded so accidental clicks can't wipe data.
- Sidebar has a direct link to the new Pre-order Tracker.
