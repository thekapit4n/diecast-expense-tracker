in `documents/changes/` so mobile and web history never mix.

## File Naming Rule

Always create one file per day using this format:

- `YYYY-MM-DD.md`

Example:

- `2026-07-19.md`

## What to Write Each Day

Keep it simple so non-technical readers can understand.

Sections per day:

1. **Quick Overview** — one-paragraph summary + the list of files touched.
2. **What Changed** — numbered points, plain language, explain the *why*.
3. **User Impact** — what the phone user can now do.
4. **Notes for Next Time** — anything to remember for the next session.
