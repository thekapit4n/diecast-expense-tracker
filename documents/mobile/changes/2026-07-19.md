# 2026-07-19 — Mobile App Kickoff (Phases 1–4)

the **same Supabase database** as the web admin, so there is no separate

## What Changed

### 1. Project setup and Supabase connection

Created the Flutter project and connected it to the existing Supabase project.
The mobile app reads the same tables the web app does, so numbers always match.
Supabase keys live in `mobile/.env`, which is **not** committed to git.

**Why:** one shared database means no data syncing to build or maintain — the
phone and the website are two windows onto the same collection.

### 2. Login and stay-signed-in

Added an email + password login using the existing Supabase account. After the
first sign-in the app stays logged in, and it protects the collection data
behind that login.

**Why:** the MVP requires personal login so only the owner sees prices and can
change data.

### 3. Home dashboard

Built the home screen summary showing: models owned, units owned, active
pre-orders, pre-orders ready to collect, unpaid/partially-paid orders, and a
list of recent purchases. It has pull-to-refresh and a big **Scan Diecast**
button. Verified live numbers against the database (207 models, 354 units, 18
active pre-orders, 8 ready to collect, 20 unpaid).

**Why:** gives an at-a-glance picture of the collection the moment the app
opens. The owned/pre-order rules are copied exactly from the web app so the two
never disagree.

### 4. Mobile catalog, search and item detail

Built an image-first catalog: two-column grid, brand filter, owned/pre-order/
not-owned filters, chase and case badges, scale tag, search by name or item
number, tap-to-open item detail (image gallery, owned/pre-order counts, and
purchase history), and pull-to-refresh. Product photos load directly from the
Supabase storage bucket as the signed-in user.

**Why:** lets the user browse and check the collection on the phone without the
web admin. Chase and normal versions show as separate tiles, matching the web.

### 5. Smart scanner (camera + barcode + OCR)

Built the scanner screen: a live barcode camera, plus "Capture text" and
"Gallery" buttons that read the box with on-device text recognition, pull out
an item number like `MGT00012`, and match it against the collection. The result
screen shows the matches and a clear **duplicate warning** when the item is
already owned or on pre-order. If nothing matches, it says so and suggests a
manual search.

**Why:** this is the headline MVP feature — point at a box in a shop and
instantly see whether it's already in the collection.

### 6. Bottom navigation

Added a bottom navigation bar to switch between **Home** and **Catalog**, each
keeping its own state.

## User Impact

On the phone, the user can now:

- Sign in with their existing account and stay signed in.
- See a live summary of their collection on the home screen.
- Browse the catalog with images, filter by brand and ownership, search, and
  open any item to see its details and purchase history.
- Open the scanner (camera/OCR features work on a real iPhone).

## Notes for Next Time

- **Google ML Kit (OCR) cannot run on the Apple Silicon iOS Simulator** (needs
  arm64, which the library doesn't ship for simulators). The scanner therefore
  must be tested on a real iPhone. Login/dashboard/catalog were all verified on
  the simulator before ML Kit was added.
- **Running on a physical iPhone** was set up but paused: the device paired with
  Xcode successfully, but Apple ID sign-in (for code signing) is pending an
  Apple ID password reset. Resume at: Xcode → Runner target → Signing &
  Capabilities → select Personal Team, then `flutter run` to the device.
- Storage images load from the authenticated endpoint
  `{SUPABASE_URL}/storage/v1/object/authenticated/diecast-images/{path}` using
  the logged-in session token (the bucket is not public).
- Still to do (later phases): quick purchase / pre-order entry forms, the
  pre-order tracker with quick status updates, and full on-device testing.
