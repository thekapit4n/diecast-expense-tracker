## What Changed

### 1. Runs on a physical iPhone (Phase 7)

Set up code signing (personal Apple ID / Team) and ran the app on a real device
in **release mode** so it launches standalone from the home screen. On-device is
required because Google ML Kit (the scanner OCR) cannot run on the Apple Silicon
simulator.

### 2. Pre-order Tracker layout fix

Fixed a device-only layout bug where long shop names rendered one letter per
line and item names overflowed. All the tracker's text is now single-line with
ellipsis, and the seller reference is flexible.

### 3. Per-order total

Each order card now shows a footer with the number of items and the order's
total value.

### 4. Bulk "mark as collected" on the Ready tab

The Ready tab now supports selecting multiple items (per-item checkboxes +
select-all) and collecting them together. Confirmation shows an itemised list,
the total, and a "How did you pay for this visit?" payment-method picker — each
selected item is marked paid in full and collected today. Mirrors the web
BulkCollectDialog.

### 5. Total pre-orders summary

The tracker summary is now a 2×2 layout with a **Total pre-orders** figure
(pending items not yet collected) alongside Balance due, Awaiting and Ready.

### 6. Navigation: Scan in the bottom menu

Added a **Scan** destination to the bottom navigation (Home · Catalog · Scan ·
Pre-orders); tapping it opens the scanner without keeping the camera alive in
the background. Removed the now-redundant "Scan Diecast" button from the home
dashboard.

### 7. Add Purchase hidden (view-only for now)

Hid the mobile "Add Purchase" button behind a feature flag
(`kEnableMobileAddPurchase`, currently false). Creating new items and adding
purchases stays on the web admin for now; flip the flag to re-enable later.

## User Impact

On the phone, the user can now:

- Run the whole app on their own iPhone.
- Manage the pre-order lifecycle, including collecting several items at once with
  a payment method, and see totals per order and overall.
- Reach the scanner from the bottom menu.

## Follow-up (same day) — branding and offline handling

### 8. App icon

Added the app icon (a diecast-in-display-case design). The artwork is stored in
the repo at `mobile/assets/icon/app_icon.png` and all 21 iOS sizes are generated
with `flutter_launcher_icons` (`dart run flutter_launcher_icons`).

The original artwork had a black margin around a rounded tile, which made the
icon look small on the home screen because iOS applies **its own** rounded mask
on top. It was cropped to full-bleed so it matches the size of other app icons;
the uncropped version is kept as `app_icon_original.png`.

**Why:** iOS always rounds icon corners itself, so icon art should fill the whole
square.

### 9. Splash screen

Added a native launch screen using the ScaleVault artwork
(`mobile/assets/splash/splash.png`) on a matching `#000816` background, generated
with `flutter_native_splash`. Because startup is fast, the splash used to flash
past before it could be read, so the app now holds it for a minimum of ~1.8
seconds before revealing the UI.

Note: `fullscreen` is deliberately kept **false** — on iOS it sets
`UIStatusBarHidden` app-wide, which would hide the clock/battery inside the app.

### 10. Renamed the app to ScaleVault

The name under the icon is now **ScaleVault** (`CFBundleDisplayName`). In-app
headings still read "Diecast Collector" and can be renamed later if wanted.

### 11. Offline / error handling (fixes the "app keeps loading" problem)

The app used to spin forever with no explanation when there was no internet.
Fixed in three parts:

- **Instant detection** — every data load (dashboard, catalog, pre-orders) and
  the login now check connectivity first (`connectivity_plus`) and fail
  immediately when the device is offline, instead of waiting.
- **Timeout backstop** — requests give up after 10 seconds, for the case where
  the device is on Wi-Fi that has no real internet.
- **A clear message** — a shared `AppErrorView` shows **"No internet connection —
  check your Wi-Fi or mobile data"** with a **Retry** button, and distinguishes
  connection problems from genuine errors ("Something went wrong"). Login shows
  the same offline wording.

**Why:** an endless spinner gives the user no idea what went wrong or what to do.

### Updated user impact

- The app has its own icon, name and splash screen.
- When offline, the user immediately sees why and can retry, instead of staring
  at a loading indicator.
